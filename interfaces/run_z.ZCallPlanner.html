order

  • order(first: ZTask<Action>, second: ZTask<Action>): void
    • Defined in run-z/run-z/src/core/plan/call-planner.ts:67

    Establishes the task execution order.

    The call to this method does not cause any of the tasks to be executed.

    Whenever a second task executed, the first one executed before it, unless they allowed to be executed in parallel.

    Contradictory execution order leads to unpredictable execution order.

    Parameters

    • first: ZTask<Action>

      The task executed first.

    • second: ZTask<Action>

      The task executed after the first one.

    Returns void
